# 公文轨范

公文轨范（Gongwen Guifan）是一款在浏览器中运行的公文排版工具。它以常见党政机关公文版式为基础，提供文稿识别、A4 分页预览、红头文件设计、格式检查和多格式导出等功能。

项目采用单文件设计，下载 HTML 文件后即可使用，不需要安装程序，也不需要配置服务器。文稿识别、排版和历史记录均在浏览器本地完成。

在线使用：<https://gw.sakuras.uk>

## 主要功能

### 文稿编辑与自动排版

- 自动识别首行标题、正文段落和多级标题
- 支持“一、”“（一）”“1.”“（1）”等常见层级
- 支持“一是、二是、三是”等叙述式层级
- 快速插入标题序号、落款和分页标记
- 清理从 Markdown 或其他编辑器复制而来的格式符号
- 实时统计字数、段落数和预估页数
- 支持撤销与重做

### A4 逐页预览

- 按实际 A4 页面尺寸生成分页预览
- 实时显示页边距、字体、字号和行距调整结果
- 支持人工指定段落从新页开始
- 附件内容可自动另页排版
- 支持日间模式和夜间模式

### 公文格式设置

默认版式参考常用党政机关公文排版参数，可根据实际需要调整：

- 页面上、下、左、右边距
- 正文、标题及各级标题字体
- 正文和标题字号
- 固定行距
- 段后间距
- 标题及标题层级自动识别

字体显示效果取决于当前设备已经安装的字体。未安装方正小标宋、仿宋_GB2312 等字体时，浏览器会自动使用候选字体替代。

### 红头文件设计

- 设置发文机关标志
- 设置发文字号
- 调整红头字体、字号和字形比例
- 设置红色分隔线
- 设置签发人、份号、密级和紧急程度
- 设置主送机关、抄送机关和印发机关
- 生成末页版记
- 在普通公文版式和红头文件版式之间切换

### 检查与校对

工具箱内置以下辅助功能：

- 格式体检
- 公文要素一致性核验
- 查找与替换
- 版本对比
- 序号校正
- 深度校对
- 模板变量填写
- 公文结构导航
- 附件生成
- 分页控制

这些功能用于发现常见的格式和结构问题，不能代替人工审核。正式印发前，仍应按照本单位的办文制度和最新规范进行复核。

### 模板与历史记录

- 内置通知、请示、报告、函、纪要和红头文件模板
- 支持保存自定义模板
- 自动保存当前文稿
- 保存本地历史版本
- 支持搜索、收藏、标签、重命名、对比和回收站
- 支持批量导出历史文稿
- 支持备份和恢复本地数据
- 工具顺序和常用门户可自行调整

### 导入与导出

支持导入：

- TXT
- Markdown
- DOCX
- 网页格式 DOC

支持导出或输出：

- DOCX
- DOC
- TXT
- PDF
- 浏览器打印

不同办公软件对字体、行距和分页的处理可能存在差异。需要正式印制的文件，建议导出后再使用目标办公软件检查一次分页和字体替换情况。

## 使用方法

### 直接使用

下载仓库中的 HTML 文件，使用现代浏览器打开即可。

```text
公文轨范.html
```

无需安装依赖，也无需启动本地服务。

### 在线部署

这是一个静态单页应用，可以部署到 GitHub Pages、Cloudflare Pages、Vercel、Netlify 或普通静态网站空间。

最简单的部署方式，是将 HTML 文件重命名为 `index.html` 后放到站点根目录。

```text
/
└── index.html
```

### 本地开发

项目的 HTML、CSS 和 JavaScript 均写在同一个文件中。修改后刷新浏览器即可查看效果。

```bash
git clone <你的仓库地址>
cd <仓库目录>
```

随后直接打开 `index.html`，或使用任意静态文件服务器运行：

```bash
python3 -m http.server 8080
```

浏览器访问：

```text
http://localhost:8080
```

## 文稿输入约定

为了获得更稳定的识别结果，建议按以下方式整理原稿：

```text
关于开展某项工作的通知

各有关单位：

一、工作目标

正文内容……

（一）重点任务

正文内容……

1. 具体事项

正文内容……

某某单位
2026年7月14日
```

一般情况下：

- 第一行会被识别为公文标题
- 空行用于分隔标题、主送机关、正文和落款
- 各级标题应单独成段
- 使用“【分页】”可在对应位置插入人工分页
- 使用“【变量名称】”可建立待填写的模板变量

## 数据与隐私

公文内容、排版设置、自定义模板和历史记录保存在当前浏览器的本地存储中，核心排版过程不依赖服务器处理。

请注意：

- 清除浏览器网站数据可能会删除本地文稿和设置
- 更换浏览器、设备或浏览器用户配置后，原有数据不会自动迁移
- 建议定期使用“数据备份”功能保存重要内容
- 公共电脑不适合长期保存涉密或敏感文稿
- 本工具不应被视为涉密信息系统

## 浏览器兼容性

建议使用较新版本的以下浏览器：

- Google Chrome
- Microsoft Edge
- Mozilla Firefox
- Safari

部分导出、打印和本地文件读取功能依赖现代浏览器接口。旧版浏览器或内置浏览器可能无法完整使用全部功能。
